02

ci.yml — kiểm thử & gate

Trigger pull_request + push. Concurrency cancel-in-progress theo ref. Job changes (dorny/paths-filter) quyết định job nào chạy — phần không đổi thì skip (skip = pass cho required check).

core Go backend

  • Go 1.25.x (patch mới nhất) + cache module
  • Postgres 16 service container (pgvector/pgvector:pg16), tạo blog_test bằng psql
  • gofmt -l · go vet · go build · go test -p 1
  • govulncheck ./... — quét vuln Go theo call-graph

web · admin Frontend

  • pnpm 11.10 + Node 22 (cache pnpm)
  • pnpm turbo run lint test build --filter=@ultimate/<app>...
  • web: ESLint 9 flat + vitest + Next 16 build (Turbopack)
  • admin: tsc --noEmit + vitest + vite build

security Trivy fs scan

  • Quét toàn repo: vuln (go.sum + pnpm-lock), misconfig (Dockerfile/compose), secret
  • Severity HIGH,CRITICAL · ignore-unfixed · exit-code 1 (blocking)
  • Luôn chạy (không path-gate)

changes paths-filter

  • services/core/** → core
  • apps/web/** + shared → web
  • apps/admin/** + shared → admin
  • shared = packages/**, lockfile, turbo.json, ci.yml

04

deploy.yml — CD lên VPS DORMANT

Chỉ workflow_dispatch (khối auto-trigger comment sẵn). Bật khi có server.

Các bước (SSH tới VPS)

  • docker compose up -d postgres
  • Migrate: one-shot arigaio/atlas container chạy migrate apply — tách khỏi entrypoint core, tránh race nhiều replica
  • compose -f prod -f deploy pull → up -d
  • Retry healthcheck curl /healthz

docker-compose.deploy.yml

Override chồng lên prod file — bỏ build:, dùng image GHCR:

# override cho core/web/admin
build: !reset null

Cần làm khi kích hoạt CD: điền secrets VPS_HOST / VPS_USER / VPS_SSH_KEY (+ GHCR_PAT nếu package private) · bỏ comment auto-trigger · điền vars.* public URL để web/admin build.

08

Bước kế tiếp — kích hoạt CD

  1. Mua/dựng VPS chạy Docker + copy .env.prod (chứa secret DB/OAuth/storage) + docker-compose.prod.yml + docker-compose.deploy.yml + services/core/migrations lên server.
  2. Có domain → điền repo Variables NEXT_PUBLIC_MEDIA_HOST + ADMIN_API_URL → push main để build+push image web/admin.
  3. Điền secrets VPS_* + (nếu package private) GHCR_PAT.
  4. Bỏ comment khối auto-trigger trong deploy.yml (hoặc chạy tay tab Actions).
  5. Verify deploy live: healthcheck /healthz xanh, blog + admin phục vụ image mới.
Nợ kỹ thuật đã note: web thiếu NEXT_PUBLIC_API_URL build-arg (beacon ViewTracker) · image FE phải rebuild mỗi env (Next static — public URL baked) · downtime ngắn khi up -d recreate (chưa zero-downtime) · CD chưa verify live (chờ VPS).
Còn thiếu trước CD live (chưa có trong compose): reverse proxy + TLS (Caddy auto Let's Encrypt) + domain + firewall chỉ mở 80/443 + backup pg_dump→R2 cron → gom vào Slice 16 (Production edge).

VPS — cấu hình khuyến nghị (0.1% expert)

Footprint idle đo thật: app (core+web+admin+postgres+redis) ~130MB, +observability ~170MB. RAM là ràng buộc chính, không phải CPU — build image chạy ở CI/GHCR, VPS chỉ pull + run. Headroom cho Postgres buffers, Next SSR burst, AI worker Python tương lai.
